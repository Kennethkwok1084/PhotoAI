# PhotoAI 开发文档索引

本目录是 PhotoAI 后续开发的主文档集，面向个人开发执行使用。旧版文档保留在 `docs/` 根目录，用作历史草案和参考。

## 技术决策

| 模块 | 决策 |
|---|---|
| 后端 | FastAPI + SQLAlchemy 2.x + Alembic |
| 数据库 | PostgreSQL 16 + pgvector |
| 队列 | Redis + Dramatiq 或 Celery，MVP 推荐 Dramatiq |
| 图片处理 | libvips 优先，Pillow 作为兼容补充 |
| 视频处理 | FFmpeg |
| AI 推理 | ONNX Runtime CPU 起步，后续接 OpenVINO / CUDA |
| Android | Kotlin + XML/ViewBinding + RecyclerView + Paging 3 |
| Android 后台 | WorkManager + Foreground Service |
| Android 本地库 | Room + OkHttp + Coil 或 Glide |
| Web | 暂不定技术栈，MVP 不阻塞 Android 与后端 |
| 部署 | Docker Compose |

Android 不采用 Jetpack Compose 作为 MVP 主方案。原因是当前目标强调最高性能和最大兼容性，传统 Android View 体系在低中端设备、长列表图库、多选、图片预览、后台稳定性上更可控。

## MVP 主线

```text
后端基础框架
→ 数据库迁移
→ 用户登录
→ Android 登录
→ Android 扫描 MediaStore
→ 分片上传
→ 服务端入库
→ 缩略图与 EXIF
→ Android 相片列表
→ AI 任务队列
→ 人脸 / OCR 基础能力
```

## 开发优先级

| 优先级 | 内容 | 目标 |
|---|---|---|
| P0 | 后端 + 数据库 | 能创建用户、登录、入库资产、上传文件 |
| P0 | Android | 能登录、扫描本机照片、上传、查看备份状态 |
| P1 | 媒体处理 | 缩略图、预览图、EXIF、视频封面 |
| P1 | AI 任务 | 任务创建、领取、失败重试、结果入库 |
| P2 | 人脸与 OCR | 完成基础识别和搜索 |
| P3 | Web | 技术栈待定，后续作为管理和浏览端 |
