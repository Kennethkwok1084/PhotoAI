| 类型 | 方案 |

## 工程约定

1. 所有数据库表使用 UUID 主键。
2. 所有时间字段使用 `TIMESTAMPTZ`。
3. 所有用户数据查询必须过滤 `owner_user_id` 或 `user_id`。
4. 文件路径只保存相对路径，不把绝对路径暴露给客户端。
5. 原图访问通过鉴权接口返回文件流或短期签名地址。
6. AI 结果必须保存来源、模型、版本、置信度。
7. 后台任务必须可重试，失败原因必须可查询。
8. 删除默认软删除。

## 认证模块

### 必做接口

```text
POST /api/auth/register
POST /api/auth/login
POST /api/auth/refresh
POST /api/auth/logout
GET  /api/auth/me
```

### Token 规则

| Token | 保存位置 | 过期 |
|---|---|---|
| access token | Android EncryptedSharedPreferences | 30 分钟 |
| refresh token | Android EncryptedSharedPreferences | 30 天 |

Refresh token 服务端需要可撤销。MVP 可先使用 `refresh_tokens` 表，后续再支持设备管理。

## 上传模块

### 服务端流程

```text
create session
→ upload part
→ record upload_parts
→ complete session
→ verify chunk count
→ merge temp file
→ verify full hash
→ move to storage
→ create asset
→ create asset_files
→ enqueue metadata / thumbnail / ai tasks
```

### 去重规则

1. 同一用户同一 `file_hash` 视为同一资产。
2. 不同用户同一 hash 可以各自拥有独立资产记录。
3. MVP 不做跨用户物理文件复用，避免权限和删除复杂化。

## 资产模块

默认列表排序：

```text
taken_at DESC NULLS LAST, imported_at DESC
```

查询必须包含：

```text
owner_user_id = current_user.id
deleted_at IS NULL
hidden = false
```

## 后台任务

任务创建时写入 `ai_tasks` 或媒体任务表。为了简化，MVP 可统一使用 `ai_tasks` 承载：

```text
thumbnail
metadata
face_detect
ocr
```

Worker 只通过 API 或队列领取任务，不直接假设业务上下文。

## 错误码

| code | 含义 |
|---|---|
| UNAUTHORIZED | 未登录或 Token 无效 |
| FORBIDDEN | 无权限 |
| VALIDATION_ERROR | 参数错误 |
| USER_EXISTS | 用户已存在 |
| INVALID_CREDENTIALS | 账号或密码错误 |
| UPLOAD_SESSION_NOT_FOUND | 上传会话不存在 |
| UPLOAD_PART_MISSING | 分片缺失 |
| HASH_MISMATCH | 文件 hash 不一致 |
| ASSET_NOT_FOUND | 资产不存在或无权访问 |
| STORAGE_UNAVAILABLE | 存储不可用 |
| TASK_NOT_FOUND | 任务不存在 |
