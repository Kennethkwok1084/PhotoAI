# PhotoAI Android 前端设计文档

## 1. 文档说明

本文档基于当前 PhotoAI Android 端 UI 设计稿编写，主要用于指导 Android 客户端前端开发、页面拆分、组件复用和交互实现。

当前设计方向为：

> 简洁、轻量、接近国内手机厂商相册风格，整体参考 HyperOS 一类的清爽卡片式设计，可适当使用毛玻璃、柔和阴影、圆角卡片和低饱和渐变。

PhotoAI Android 端不是传统网盘文件管理器，而是一个以“相册体验 + AI 发现 + 私有备份”为核心的移动相册应用。

---

## 2. 设计目标

### 2.1 核心目标

1. 用户打开 App 后能快速知道照片是否正在备份；
2. 用户可以像系统相册一样直接浏览所有照片；
3. 用户可以通过时间轴查看更有故事感的照片分组；
4. 用户可以在“发现”页完成 AI 搜索、人物、地点、标签、智能推荐等功能；
5. 用户可以在“我的”页管理备份、存储、隐私、安全和 AI 设置；
6. 整体视觉应接近手机厂商相册，而不是后台管理系统。

### 2.2 设计关键词

```text
简洁
通透
轻卡片
毛玻璃
低干扰
圆角
留白
AI 感
手机厂商风格
```

## 3. 底部导航设计

### 3.1 导航结构

底部 TabBar 只保留以下 5 个入口：

```text
首页
相片
时间轴
发现
我的
```

不要再单独放“人物”“AI 搜索”“备份”“设置”等入口。  
这些功能应归入对应页面中。

| Tab | 页面 | 作用 |
|---|---|---|
| 首页 | Home | 总览、备份状态、智能记忆、常用人物、快捷入口 |
| 相片 | Photos | 所有照片的普通图库浏览页面 |
| 时间轴 | Timeline | 以日期和事件卡片组织的故事流 |
| 发现 | Discover | 合并人物、地点、标签、AI 搜索、智能推荐 |
| 我的 | Profile | 个人中心、备份、存储、安全、设置 |

### 3.2 TabBar 视觉

建议采用半透明毛玻璃底栏：

```text
高度：64dp - 72dp
背景：白色 80% 透明度 + blur
圆角：顶部 20dp 或整体胶囊式 24dp
阴影：轻微投影
选中态：主色图标 + 主色文字
未选中态：灰色图标 + 灰色文字
```

图标风格应保持线性、圆润、轻量。

建议图标：

| Tab | 图标建议 |
|---|---|
| 首页 | home |
| 相片 | image / photo |
| 时间轴 | clock / calendar |
| 发现 | sparkle / search / compass |
| 我的 | user |

## 4. 全局视觉规范

### 4.1 色彩

主色建议使用蓝紫色系，贴近现代手机系统风格。

```text
Primary：#4F6BFF
Primary Gradient：#5C7CFF → #8D6BFF
Background：#F6F8FC
Card：#FFFFFF / rgba(255,255,255,0.78)
Text Primary：#121826
Text Secondary：#6B7280
Text Tertiary：#9CA3AF
Border：#E8ECF3
Success：#22C55E
Warning：#F59E0B
Error：#EF4444
```

### 4.2 毛玻璃效果

用于以下区域：

- 底部导航栏；
- 搜索框背景；
- 首页智能卡片；
- 照片详情页底部操作栏；
- 时间轴事件卡片浮层；
- 发现页 AI 搜索框。

实现建议：

```text
背景透明度：70% - 90%
模糊半径：16dp - 24dp
边框：1px 半透明白色
阴影：低透明度柔和阴影
```

Android 原生实现时，如果系统版本或性能不适合实时 blur，可以使用半透明背景 + 柔和阴影作为降级方案。

### 4.3 圆角

整体圆角应统一，避免混乱。

| 组件 | 圆角 |
|---|---|
| 小按钮 | 10dp - 12dp |
| 搜索框 | 18dp - 24dp |
| 普通卡片 | 18dp - 22dp |
| 图片缩略图 | 8dp - 12dp |
| 大图卡片 | 20dp - 28dp |
| 底部导航 | 24dp |

### 4.4 字体层级

Android 可使用系统默认字体，中文以系统字体为准。

| 用途 | 字号 | 字重 |
|---|---|---|
| 页面标题 | 22sp - 26sp | SemiBold |
| 区块标题 | 16sp - 18sp | Medium |
| 正文 | 14sp - 15sp | Regular |
| 辅助文字 | 12sp - 13sp | Regular |
| 标签文字 | 12sp | Medium |

## 5. 页面一：首页

### 5.1 页面定位

首页是用户进入 App 后的总览页，重点展示：

1. 当前备份状态；
2. 最近记忆或智能推荐；
3. 常用人物；
4. 快捷入口；
5. 最近照片或智能卡片。

首页不应堆太多功能，主要负责“让用户快速知道相册整体情况”。

### 5.2 页面结构

```text
顶部状态栏
页面标题 PhotoAI
通知按钮
搜索框
备份状态卡片
智能记忆卡片
常用人物
常去地点 / 快捷入口
底部导航栏
```

### 5.3 组件说明

#### 5.3.1 顶部标题区

内容：

```text
PhotoAI
右上角通知图标
```

要求：

- 标题靠左；
- 通知图标靠右；
- 不要使用复杂顶部栏；
- 页面整体保持系统相册风格。

#### 5.3.2 搜索框

占位文案：

```text
搜索照片、地点、人物、标签...
```

右侧可放语音或筛选图标。

点击后可跳转到“发现”页的搜索状态，也可以进入全局搜索页。

#### 5.3.3 备份状态卡片

显示内容：

```text
备份中
23.6GB / 64GB
68%
进度条
```

状态可分为：

```text
正在备份
已完成备份
等待 Wi-Fi
等待充电
备份暂停
备份失败
```

交互：

- 点击进入“我的 > 备份 / 上传”；
- 如果有失败项，显示红色轻提示；
- 如果已完成，显示绿色完成状态。

#### 5.3.4 智能记忆卡片

示例：

```text
周末露营记
2024年5月12日 · 32张照片
查看回忆
```

视觉：

- 大图背景；
- 左下角文字浮层；
- 毛玻璃按钮；
- 卡片圆角较大。

#### 5.3.5 常用人物

显示 4 - 5 个常见人物头像：

```text
宝宝
妈妈
爸爸
爷爷
奶奶
```

点击人物进入“发现 > 人物详情”。

#### 5.3.6 快捷入口

可展示：

```text
相册
时间轴
发现
释放空间
```

MVP 阶段不需要太多，避免首页变复杂。

## 6. 页面二：相片

### 6.1 页面定位

“相片”页面是最传统、最直接的图库浏览页面，用于展示用户所有照片和视频。

它与“时间轴”的区别：

```text
相片：普通图库，重点是快速浏览全部照片
时间轴：故事化时间流，重点是按事件和回忆组织照片
```

### 6.2 页面结构

```text
页面标题：相片
搜索图标
筛选图标
分类 Tab：全部 / 视频 / 收藏 / 截图
月份分组
照片宫格
底部导航栏
```

### 6.3 顶部区域

标题：

```text
相片
```

右侧图标：

```text
搜索
筛选
```

搜索：进入全局搜索。  
筛选：弹出筛选面板。

### 6.4 分类 Tab

建议分类：

```text
全部
视频
收藏
截图
```

后续可增加：

```text
RAW
动图
下载
相机
微信
```

但 MVP 阶段不要放太多。

### 6.5 照片宫格

设计要求：

- 一行 3 - 4 列；
- 缩略图间距 2dp - 4dp；
- 图片圆角 4dp - 8dp；
- 支持视频角标；
- 支持收藏角标；
- 支持已备份状态可选显示；
- 支持按月份分组。

月份标题：

```text
2024年5月
2024年4月
```

### 6.6 交互

| 操作 | 行为 |
|---|---|
| 点击图片 | 进入照片详情页 |
| 长按图片 | 进入多选模式 |
| 下拉刷新 | 刷新本地与云端状态 |
| 上滑 | 加载更多 |
| 点击月份 | 可折叠该月份，后期可做 |
| 点击筛选 | 弹出筛选面板 |

### 6.7 多选模式

长按后进入多选模式：

顶部显示：

```text
已选择 3 项
取消
```

底部操作：

```text
分享
加入相册
收藏
删除
更多
```

## 7. 页面三：时间轴

### 7.1 页面定位

时间轴不是普通图库，不应该只是照片列表。  
它应该更像手机厂商相册里的“回忆流 / 故事线”。

当前设计稿的方向是正确的：用大卡片按日期和事件组织照片。

### 7.2 页面结构

```text
页面标题：时间轴
日历按钮
筛选按钮
左侧时间线节点
事件卡片列表
底部导航栏
```

### 7.3 时间轴卡片

每张卡片代表一个日期或系统识别出的事件。

示例：

```text
云南 · 泸沽湖之旅
166 张照片 · 3 个视频
2024年5月20日 周一
```

卡片内容：

- 大封面图；
- 事件标题；
- 照片数量；
- 视频数量；
- 小图预览条；
- 日期信息。

### 7.4 创意设计方向

时间轴可以做出和普通图库不同的感觉：

#### 方向一：事件故事卡

系统根据时间、地点、人物自动聚合：

```text
云南 · 泸沽湖之旅
家庭聚会
露营之夜
城市漫步
```

每个事件以横向大卡片呈现。

#### 方向二：左侧时间节点

左侧显示竖向时间线，节点对应日期。

视觉：

```text
蓝色小圆点
细线连接
当前滚动日期高亮
```

#### 方向三：智能标题

AI 根据地点、人物、场景生成标题：

```text
湖边的一天
和家人的晚餐
夏日花海
夜晚露营
```

MVP 阶段可以先使用规则标题：

```text
地点 + 日期
人物 + 场景
```

#### 方向四：卡片内小缩略图

大封面下面叠加 3 - 5 张小缩略图，让用户一眼知道这个事件包含什么。

### 7.5 时间轴与相片的区别

| 页面 | 重点 | 展示方式 |
|---|---|---|
| 相片 | 全量浏览 | 普通宫格 |
| 时间轴 | 回忆和事件 | 大卡片 + 日期节点 |
| 发现 | AI 整理 | 人物、地点、标签、推荐 |

## 8. 页面四：发现

### 8.1 页面定位

“发现”页面合并原本的“人物”和“AI 搜索”，是 PhotoAI 的智能中心。

它负责：

1. AI 搜索；
2. 常见人物；
3. 地点；
4. 标签；
5. 智能推荐；
6. 相似照片清理入口；
7. 回忆推荐。

不要再单独设置“人物”Tab 或“AI 搜索”Tab。

### 8.2 页面结构

```text
页面标题：发现
搜索框
常见人物
地点
热门标签
智能推荐
底部导航栏
```

### 8.3 搜索框

占位文案：

```text
搜索照片、地点、人物、标签...
```

搜索能力包含：

- 人物；
- 地点；
- 标签；
- OCR 文本；
- 语义搜索；
- 日期；
- 文件名。

注意：不要在 UI 上单独放“OCR 文字识别”模块。  
OCR 应该作为搜索能力存在，而不是一个独立卡片。

### 8.4 常见人物

展示 4 - 5 个常见人物头像。

内容：

```text
头像
姓名
照片数量
```

右侧“更多”进入完整人物列表。

完整人物列表可以后续作为二级页：

```text
发现 / 全部人物
```

### 8.5 地点

展示常去地点：

```text
泸沽湖
上海
三亚
更多地点
```

地点卡片使用图片背景，并展示照片数量。

### 8.6 热门标签

标签使用胶囊按钮：

```text
# 旅行
# 美食
# 家庭
# 宠物
# 露营
# 自然
# 城市
```

点击标签进入搜索结果页。

### 8.7 智能推荐

推荐卡片示例：

```text
春日花海
根据你的照片生成
68 张照片

亲子时光
识别到人物组合
52 张照片
```

后期可以加入：

```text
相似照片清理
低质量照片清理
截图整理
证件照片
发票票据
```

## 9. 页面五：照片浏览 / 照片详情

### 9.1 页面定位

照片详情页用于查看单张照片、照片信息和执行操作。

它是“相片页面”点击照片后的核心页面。

### 9.2 页面结构

```text
顶部沉浸式图片区域
返回按钮
当前序号
收藏按钮
更多按钮
照片主体
信息卡片
底部操作栏
```

### 9.3 顶部区域

显示：

```text
返回
1/23
收藏
更多
```

顶部按钮可使用半透明毛玻璃圆形按钮，保证不遮挡照片。

### 9.4 图片区域

要求：

- 默认完整适配屏幕宽度；
- 支持双指缩放；
- 支持左右滑动切换；
- 支持点击隐藏 UI；
- 视频显示播放按钮；
- Live Photo 后期可显示动态标记。

### 9.5 信息卡片

显示内容：

```text
云南 · 泸沽湖
2024年5月20日 14:32
云南省 宁蒗县 泸沽湖自然景区
Xiaomi 14 Ultra · Leica
23mm · f/1.63 · 1/1250s · ISO 50
4000 × 3000 · 3.2MB
```

按钮：

```text
编辑
查看位置
```

如果照片有人脸、OCR、标签，可在信息卡下方后续增加：

```text
人物
标签
识别文字
```

但 MVP 阶段不建议堆太满。

### 9.6 底部操作栏

固定操作：

```text
分享
收藏
编辑
删除
更多
```

底部操作栏可使用毛玻璃样式。

## 11. 全局搜索设计

虽然没有单独 Tab，但搜索是核心能力，应在首页和发现页都能进入。

### 11.1 搜索入口

入口位置：

- 首页搜索框；
- 发现页搜索框；
- 相片页右上角搜索图标；
- 时间轴页可选搜索图标。

### 11.2 搜索范围

统一搜索应支持：

```text
人物
地点
标签
OCR 文本
日期
文件名
设备型号
语义描述
```

### 11.3 搜索结果分组

建议搜索结果按类型分组：

```text
人物
地点
标签
照片
识别文字
```

OCR 不作为一级页面展示，但可以作为搜索结果的一部分。

## 18. 当前设计结论

当前 Android 端前端结构最终确定为：

```text
首页：总览和快捷入口
相片：普通照片图库
时间轴：故事化事件流
发现：人物 + 地点 + 标签 + AI 搜索
我的：个人中心和系统管理
```

其中：

- “人物”不再作为底部 Tab，归入“发现”；
- “AI 搜索”不再作为底部 Tab，归入“发现”；
- “OCR 文字识别”不再单独显示为页面模块，而是作为搜索能力；
- “相片”页面必须单独存在，用于普通图库浏览；
- “时间轴”页面要区别于相片页，采用事件卡片和故事流设计；
- “照片详情页”是从相片、时间轴、发现等页面共同进入的详情页面。

PhotoAI Android 端的前端重点不是堆功能，而是让用户感觉它像一个真正的系统级相册：打开就能看、备份状态清楚、照片浏览顺滑、AI 发现自然融入。
